# Context7 First Rule

## Mandatory Documentation Verification

Before implementing, modifying, refactoring, or generating code related to any framework, library, SDK, API, package, or tool, always perform the following process:

### Step 1: Documentation Lookup

Use Context7 to retrieve the latest official documentation and examples for:

* React
* Vite
* TypeScript
* Tailwind CSS
* Shadcn/UI
* Lucide React
* TanStack Query
* React Router
* Axios
* Node.js
* Express
* SQL Server
* Prisma
* PostgreSQL
* Any third-party package involved in the task

### Step 2: Validate Existing Project Context

Before making changes:

* Analyze the current project structure.
* Inspect existing patterns and conventions.
* Identify already installed dependencies.
* Verify architecture decisions already in place.
* Avoid introducing conflicting patterns.

### Step 3: Plan Before Coding

Always provide:

1. Current state analysis.
2. Documentation findings from Context7.
3. Implementation plan.
4. Risks or breaking changes.

Only after completing these steps should implementation begin.

### Step 4: Preserve Consistency

New code must:

* Follow existing architecture.
* Reuse existing components.
* Reuse existing hooks.
* Reuse existing services.
* Reuse existing utilities.
* Follow project naming conventions.
* Follow TypeScript strict typing.

### Step 5: Modern Standards

Always prefer:

* Latest stable APIs.
* Official recommendations from documentation.
* Performance-oriented solutions.
* Accessibility best practices.
* Security best practices.

Never rely solely on training knowledge when Context7 documentation is available.

Context7 is the source of truth.
